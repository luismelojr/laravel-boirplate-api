# Roadmap

## Objetivo da skill

Levar uma pessoa que está começando do zero em Vue/Nuxt até uma base sólida para construir aplicações Nuxt com `TypeScript-first`, entendendo bem os fundamentos antes de avançar.

## Ordem obrigatória

1. `Module 1` - Base web + TypeScript essencial
2. `Module 2` - Fundamentos de Vue com TypeScript
3. `Module 3` - Componentes Vue tipados
4. `Module 4` - Entrada no Nuxt com TypeScript
5. `Module 5` - Páginas, rotas e layouts
6. `Module 6` - Dados, SSR e hidratação
7. `Module 7` - Server routes e full-stack no Nuxt
8. `Module 8` - Composables, estado e formulários tipados
9. `Module 9` - Arquitetura real e boas práticas

## Regras de progressão

- Se o usuário não disser onde está, começar do `Module 1`.
- Não pular módulos base só porque surgiu uma dúvida avançada.
- Se a dúvida depender de um conceito ainda não ensinado, voltar um passo e reconstruir o fundamento.
- Cada módulo deve terminar com:
  - um exercício curto,
  - uma checagem rápida de entendimento,
  - e um resumo de 3 a 5 pontos.

## Critério mínimo para avançar

O aluno pode seguir quando:
- consegue explicar o conceito central com as próprias palavras,
- entende o papel do tipo principal usado no módulo,
- e consegue acompanhar ou adaptar o exemplo mínimo sem copiar cegamente.

## Ritmo da aula

Cada sessão deve usar esta sequência:

1. Objetivo
2. Explicação simples
3. Exemplo mínimo tipado
4. Exercício curto
5. Checagem de entendimento
6. Resumo
7. Ponte para o próximo módulo

## Pontos de atenção globais

- `TypeScript` não substitui validação em runtime.
- `Vue` ensina reatividade e composição de UI.
- `Nuxt` acrescenta convenções, SSR, estrutura full-stack e utilitários de framework.
- O aluno precisa aprender a diferenciar:
  - tipo em tempo de build,
  - estado reativo em runtime,
  - código que roda no servidor,
  - código que roda no cliente.

## Mapa de docs oficiais

Use docs oficiais principalmente para validar:
- `Introduction`
- `Routing`
- `Data Fetching`
- `Server`
- `Rendering` e `routeRules`

Se houver Nuxt MCP disponível, ele é o caminho preferido para essa validação.
